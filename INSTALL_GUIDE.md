## 1. 环境准备

### 1.1 检查 Python 是否已安装

按 `Win + R`，输入 `cmd`，回车打开命令提示符：

```
python --version
```

如果显示版本号（如 `Python 3.11.x` 或 `Python 3.12.x`），说明已安装 Python。

> **注意**：如果提示"不是内部命令"，需要安装 Python。

### 1.2 安装 Python（如果没有）

1. 打开 https://www.python.org/downloads/

2. 点击 **Download Python 3.12.x** 下载安装包

3. 运行安装程序，**务必勾选**：
   - ✅ **Add Python to PATH** （最重要！）
   - ✅ **Install pip**

4. 点击 **Install Now**

5. 安装完成后，**重新打开命令提示符**（关闭之前的窗口），验证：
   ```
   python --version
   ```

## 2. 安装步骤

### 2.1 下载项目

**方式一：下载压缩包（推荐新手）**

1. 打开 https://github.com/martin202008/ZJClaw
2. 点击绿色的 **Code** 按钮
3. 点击 **Download ZIP**
4. 解压到任意目录，如 `D:\ZJClaw`

**方式二：使用 Git 克隆**

如果已安装 Git，打开命令提示符：
```
git clone https://github.com/martin202008/ZJClaw.git
```

### 2.2 安装依赖

1. 打开命令提示符
2. 进入项目目录（根据你解压的位置）：
   ```
   cd D:\ZJClaw
   ```
3. 安装依赖：
   ```
   pip install -e .
   ```
4. 等待安装完成（约2-5分钟）

> 如果提示"pip 不是内部命令"，请重新安装 Python 并确保勾选 "Add to PATH"

## 3. 启动服务

### 3.1 启动 WebUI

1. 在命令提示符中进入 webui 目录：
   ```
   cd D:\ZJClaw\webui
   ```
2. 启动服务：
   ```
   python standalone_app.py
   ```
3. 看到以下信息表示启动成功：
   ```
   [ZJClaw] WebUI starting... http://localhost:5000
      Press Ctrl+C to stop server
   ```

### 3.2 访问网页

1. 打开浏览器（推荐 Chrome 或 Edge）
2. 地址栏输入：http://localhost:5000
3. 回车访问

## 4. 基本配置

### 4.1 配置 API Key

> ZJClaw 需要连接大语言模型才能工作，需要配置 API Key。

1. 获取 API Key：
   - 打开 https://platform.minimaxi.com
   - 注册/登录账号
   - 进入控制台 → API Keys → 创建新密钥
   - 复制生成的 Key（格式类似：`eyJh...`）

2. 在网页端配置：
   - 点击左侧菜单「配置」
   - 点击「AI 设置」
   - 选择提供商：**minimax**
   - 选择模型：**MiniMax-M2.7**（推荐）
   - 粘贴 API Key
   - 点击「保存设置」

### 4.2 设置工作区（可选）

工作区是智能体操作文件的默认目录，建议设置为专门存放项目文档的文件夹：

1. 点击左侧菜单「配置」
2. 点击「工作区设置」
3. 输入文件夹路径，如 `D:\ZJClaw文档`
4. 勾选「启用安全沙箱」
5. 点击「保存设置」

> 启用沙箱后，智能体只能在该文件夹内操作文件，更安全。

## 6. 常见问题

### Q1: 启动后浏览器打不开 http://localhost:5000

**解决**：
1. 确认服务已启动，窗口显示 "Running on http://localhost:5000"
2. 尝试访问 http://127.0.0.1:5000
3. 检查浏览器代理设置，尝试关闭代理
4. 检查防火墙设置

### Q2: 提示"API Key 无效"

**解决**：
1. 确认 API Key 填写正确，没有多余空格
2. 确认 API Key 还有效（未过期）
3. 尝试重新创建新的 API Key

### Q3: 回复很慢或无响应

**解决**：
1. 检查网络连接
2. 等待片刻，大模型响应需要时间
3. 尝试切换其他模型

### Q4: 如何更新到新版本？

**解决**：
1. 点击左侧「系统」→「版本」
2. 点击「检查更新」
3. 如果有新版本，点击「一键更新」
4. 重启服务

### Q5: 文件操作失败

**解决**：
1. 如果启用了工作区沙箱，确认文件在沙箱目录内
2. 检查文件夹是否有写入权限
3. 路径中避免使用中文或特殊字符
